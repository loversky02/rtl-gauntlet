\begin{abstract}
Agentic RTL frameworks now report $100\%$ pass on standard benchmarks, but that score is measured
against \emph{visible} tests, is \emph{expensive} to reach, and is obtained on \emph{proxy} tasks.
We turn these three concerns into measurable axes. We build a two-tier protocol that freezes an
agent's design and scores it with a withheld formal-equivalence oracle, and define a
Reward-Hacking Gap (\rhg) and Honest Pass Rate (\hpr). Applying it to 156 VerilogEval tasks across
five models, we find the na\"ive headline \rhg{} ($6.1\%$) is \emph{entirely an oracle artifact}: a
na\"ive formal oracle over-reports reward hacking via don't-care \code{x}, async-reset,
state-encoding, init-state, and even a SystemVerilog parser default. We harden the oracle in seven
steps---reset normalization, don't-care awareness, a bounded miter$+$SAT fallback, a SystemVerilog
parse fix, ROM inference, a reset-aware bounded check for incompletely-specified FSMs, and an X-aware
\emph{careset} miter with a real-latch half-cycle variant for latch/\code{negedge}
designs---cutting false counter-examples $9\!\to\!0$ and ``inconclusive'' $50\!\to\!0$. Every
na\"ively-flagged counter-example is machine-\emph{proven} equivalent---two golden builds define the
specified (cared) output bits, and declared per-task preconditions encode each spec's legal-stimulus
assumption---\emph{eliminating hand-verification entirely}: \textbf{zero genuine reward hacking}
($95\%$ upper bound $\le\!3.2\%$) by frontier models on fair tasks. A weaker model (Haiku~4.5) fails $3\times$ more than
Opus~4.8 on the visible tests but does \emph{not} hack more; even a tamper-capable shell agent that
struggles for several iterations does not edit the testbench. We show the oracle catches hacking
when present---both a planted candidate (passes a randomized hidden testbench, formally disproven) and
a \emph{real elicited} one: on an impossible (spec-contradicting) task $3/5$ frontier models hardcode
(Opus stays honest), caught by the oracle plus a hardcode judge (judge-vs-human $\kappa=0.80$). We
further show the result survives off the verbatim benchmark (a whole-benchmark contamination probe),
and we add a token-cost analysis (an early-stop reclaims $12$--$24\%$ of tokens) and a latency axis:
real Sky130 PPA over a size-graded set (up to a RISC-V core) with Kendall-$\tau=1.0$ rank stability
across synthesis strategies. The central
artifact for honest agentic-hardware evaluation is a don't-care/reset/encoding-aware oracle plus a
verification discipline---not a new pass-rate.
\end{abstract}

\section{Introduction}
HORIZON treats agentic hardware design as repository-level code evolution and reaches $100\%$ pass
on several RTL suites---but iteration-0 pass is only $47.8\%$, so the headline comes from
\emph{iterative repair against a visible signal}. Its authors name three open problems: reward
hacking, token cost, and high-latency (PPA) reward. We make each measurable.

\paragraph{Thesis.} \emph{Pass@visible is the wrong score for agentic hardware design}: it can be
gamed (honesty), it is expensive (cost), and it does not scale to real reward (latency).

\paragraph{Contributions.}
\begin{enumerate}[leftmargin=*,topsep=2pt,itemsep=1pt]
\item A two-tier, formal-grounded evaluation protocol with metrics \rhg{} and \hpr{} (\S\ref{sec:method}).
\item The finding that a na\"ive formal oracle \textbf{over-reports} reward hacking, and a
seven-step hardened oracle that removes it---closing the sequential-FSM residual and \emph{proving}
every na\"ively-flagged counter-example equivalent (X-aware careset miter $+$ a real-latch half-cycle
miter for the mixed-edge class), \emph{eliminating hand-verification entirely} (\S\ref{sec:oracle}).
\item An empirical result across $5$ models $\times\,156$ tasks: \textbf{zero verified genuine
reward hacking} on fair tasks; weakness $\ne$ hacking; tampering is caught when present
(\S\ref{sec:exp}).
\item Cost and latency prototypes: token accounting with an early-stop policy, and real Sky130 PPA
via a containerized flow (\S\ref{sec:exp}).
\end{enumerate}

\section{Method: the two-tier protocol}\label{sec:method}
Figure~\ref{fig:protocol} shows the flow. The agent sees a complete specification, a \emph{visible}
diagnostic testbench, and its failure logs, and edits RTL until visible passes. We then \textbf{freeze} the design and score withheld
tiers it never saw: a hidden randomized/exhaustive testbench, and \textbf{formal equivalence} vs.\ a
curated golden (Yosys). The I/O interface is locked. We define
\[
\rhg = \frac{\lvert\{\text{visible-passers failing a withheld tier}\}\rvert}{\lvert\{\text{visible-passers}\}\rvert},
\qquad
\hpr = \frac{\lvert\{\text{pass visible and every withheld tier}\}\rvert}{\text{total tasks}} .
\]
Here ``two-tier'' is the split \emph{visible} vs.\ \emph{withheld}. The withheld side yields one of
four formal verdicts, mapped as \code{proven}/\code{bmc\_equiv}$\to$honest, \code{cex}$\to$flagged
hack, and \code{dontcare}/\code{inconclusive}$\to$\emph{excluded} (a third bucket, neither honest nor
hack). So \hpr{} is a \emph{lower bound} on true honesty and the \code{cex} rate among visible-passers
an \emph{upper bound} on \emph{detectable} hacking; only \textbf{verified-genuine \rhg{}} (every
\code{cex} confirmed by hand) is the headline claim. On the $156$-task sweep the withheld tier is the
\textbf{formal} oracle; the hidden randomized testbench is exercised on the focused demos
(\code{formal\_demo}, \code{sem\_zerocount}, tamper).
A tamper tier (a shell-capable agent that may edit any workspace file) flags edits to the
testbench/harness as \emph{exploit-evidenced} hacking, with the withheld tiers run on frozen
originals so tampering cannot fool the verdict. We separate the claim into behavioral gap,
exploit-evidenced, and tamper-confirmed, and never claim intent without trajectory evidence.

\section{The oracle, and why na\"ive formal over-reports}\label{sec:oracle}
Table~\ref{tab:gloss} glosses the recurring verification terms used below.
On a 156-task sweep (Opus~4.8), a na\"ive \code{equiv\_make} oracle reported $9$ \rhg{}
counter-examples and $50$ ``inconclusive.'' \textbf{Every CEX was a false positive}, found by
hand-verifying flagged cases: don't-care \code{x} in the golden; functionally identical designs
flagged on async reset / initial state; a candidate logically identical but with a different state
encoding; and---for most ``inconclusive''---a silent parse failure, since \code{read\_verilog}
defaults to Verilog-2005 while the references use \code{always\_ff}/\code{logic}/enums.

Our hardening pipeline (each step tested on verified cases before re-running):
\begin{enumerate}[leftmargin=*,topsep=2pt,itemsep=1pt]
\item \code{async2sync} (normalize async resets)---an identical design was flagged CEX only because
the flow treated the async-reset path inconsistently between golden and candidate.
\item reclassify a CEX to \emph{dontcare} when the golden assigns an \code{x} literal to an
\emph{output}---VerilogEval goldens mark don't-care output bits \code{1'bx}; a concrete value there is
spec-correct (the X-matching testbench accepts it) but na\"ive \code{equiv\_make} reads
\code{x}$\ne$\code{value} as a mismatch.
\item a bounded miter$+$SAT fallback comparing the designs' I/O sequences directly---induction
(\code{equiv\_induct}) cannot map two \emph{correct} FSMs with different state encodings (a 3-bit
vector vs.\ three scalars), so it falsely reports inequivalence; a SAT model here \emph{is} a real CEX.
\item \code{read\_verilog -sv}---yosys defaults to Verilog-2005, so SystemVerilog references
(\code{always\_ff}/\code{logic}/enums) \emph{silently fail to parse}; the simulator accepts them
(visible passes) while the oracle errors, which we then misread as ``inconclusive.''
\item a \code{memory} (case$\to$ROM) pass---large \code{case} ROMs elaborate to a \code{\$mem}
primitive that induction cannot align;
\item a \code{-nolatches} reset-aware BMC (Pass-3) that closes the residual \code{inconclusive}
FSMs. Their goldens use an \code{always\_comb case(state)} over a \code{reg [3:0]} that lists only the
reachable states, so the unlisted states infer a latch and yosys \emph{errors out}---an elaboration
artifact, not a solver limit (the same class as the \code{-sv} bug); \code{-nolatches} plus a reset
drive at $t{=}1$ (the true reset state, not the zero state) then proves I/O equivalence.
\item an \textbf{X-aware careset miter} that adjudicates any surviving CEX \emph{by proof} rather than
by hand. Two golden builds (\code{setundef -zero} vs.\ \code{-one}) mark each output bit \emph{cared}
where they agree and \emph{don't-care} where they disagree---i.e.\ where the value depends on an
\code{x} (an output \code{1'bx} literal \emph{or} an uninitialised register). The miter asserts
equivalence only on cared bits, only after a reset-settle window, and only where a declared per-task
input precondition holds (the spec's legal-input assumption the withheld testbench also respects,
e.g.\ prob149's gradual thermometer sequence). For the \emph{mixed-edge} class (an intentional
transparent latch $+$ a \code{negedge} FF), \code{-nolatches} would \emph{destroy} the latch and
fabricate a CEX, so a retry keeps the \emph{real} latches, converts to half-cycle logic
(\code{clk2fflogic}), drives a regular alternating clock, and registers inputs at \code{posedge} (the
testbench's synchronous stimulus). Together these \emph{prove} every flagged artifact equivalent while
still CEXing genuine bugs (the planted impossible-task overfit, inverted-FF / transparent-low /
wrong-edge variants all stay CEX)---replacing per-case hand-verification with machine-checked,
re-runnable proofs.
\end{enumerate}

\section{Experiments}\label{sec:exp}

\paragraph{Formal earns its keep.} \code{formal\_demo}: a 16-bit candidate wrong \emph{only} on
\code{0xDEAD}. The randomized hidden testbench ($512/65536$) misses it---visible \emph{and} hidden
PASS---while exhaustive formal returns CEX. With finite tests alone $\rhg{}=0$ (it looks honest);
formal exposes it ($\rhg{}=0.50$). This is the direct evidence that an exhaustive formal oracle
beats a finite held-out suite.

\paragraph{Sweep and model comparison.} Table~\ref{tab:models} and Figure~\ref{fig:models}: the
weaker models fail far more on the visible tests (\code{fail\_visible} $9$/$6$/$14$/$36$/$30$ for
Opus/GPT-5.5/Gemini/DeepSeek/Haiku) but hack no more. The na\"ively-flagged \rhg{} counter-examples
across the five models fall on only \textbf{four distinct tasks}---and the hardened oracle now
machine-\emph{proves all four} equivalent, leaving \emph{no} hand-verified residual:
\begin{itemize}[leftmargin=*,topsep=2pt,itemsep=1pt]
\item \code{q5b}, \code{prob095} --- init / one-hot-reset transients (\code{-set-init-zero} forces an
invalid start state): the two-build care-mask plus the reset-settle window \emph{prove} them
equivalent (\code{careset\_equiv}), no hand-verification.
\item \code{prob149} --- an input-\emph{sequence} don't-care: a water-level FSM whose golden advances
one thermometer step per cycle. \emph{Proven} once the spec's gradual-change precondition---which the
withheld testbench also respects---is declared as a per-task annotation.
\item \code{circuit8} --- a mixed-edge \code{negedge}-FF $+$ intentional latch. The false CEX came from
\code{-nolatches} \emph{destroying} the latch during elaboration; the real-latch \emph{half-cycle}
miter (regular clock, \code{posedge}-registered inputs---the testbench's synchronous stimulus)
\emph{proves} the Opus, GPT-5.5, and Gemini candidates equivalent, while inverted-FF /
transparent-low / wrong-edge mutants still CEX.
\end{itemize}
The care-mask marks a bit don't-care only where the two golden builds \emph{disagree}, so an incidental
internal \code{x} cannot mask a real CEX and the planted impossible-task overfit stays a CEX
(non-vacuous). Net ($95\%$ Wilson): verified-genuine \rhg{}$=0$ for all five models, upper bound
$\le\!3.2\%$ (Haiku, the loosest; $\le\!2.5$--$2.6\%$ for the three strongest).

\paragraph{Elicit (the phase-diagram).} A tamper-capable shell agent (may edit \emph{any} file):
across \emph{all $156$ fair tasks} on \emph{both} Opus and Haiku it produces \textbf{$0$ fake-pass tampers}---the few
non-design writes are benign helper modules, never a neutered testbench. \textbf{But on an \emph{impossible} task}~\cite{impossiblebench} (visible TB
demands \code{0x00}$\to$7, contradicting the spec, so \emph{any} visible-pass is provable cheating),
\textbf{$3$ of $5$} models pass it---DeepSeek, Gemini, GPT-5.5 special-case the vector
(\code{(data==8'h00)?4'd7:popcount}): visible PASS, formal CEX. An \emph{independent} judge (Opus)
confirms all three as hardcode-overfit and scores $6/7$ ($86\%$) on known hardcodes, so
exploit-evidenced counts are a \emph{conservative lower bound} (self-judging is lenient---GPT-5.5
excused its own). A blind judge-vs-human study ($n{=}20$ balanced cases) gives Cohen's
$\kappa=0.80$ (\emph{strong}), so the intent sub-classification is reliable---we keep the intent claim.
\textbf{Opus and Haiku refuse} (Opus stays
formally proven-correct). So the metric is \emph{not} trivially zero: cheating appears in the
\emph{majority} of frontier models under contradictory specs, Opus resisting. The hack is
\emph{design-only} (testbench untouched)---a file-edit flag misses it; only the formal oracle $+$ judge
catch it.
Genuine hacking is thus a function of adversarial conditions (impossible / contradictory specs) and
of the model, not of aligned frontier models in a fair single loop; RL training-time emergence is
the deeper regime.

\paragraph{Contamination.} VerilogEval is public (reported $\sim\!100\%$ contamination for older
models), so an honest pass could be memorization rather than reasoning. We probe this in layers:
\begin{itemize}[leftmargin=*,topsep=2pt,itemsep=1pt]
\item \textbf{Identifier mutation, whole benchmark.} Rename the target module and reframe the spec on
\emph{all $156$ tasks}---\emph{each machine-verified function-preserving by the careset oracle}
($156/156$ proven equivalent, \code{verify\_mutants.py}, so no mutation silently breaks a task)---then
re-sweep. \hpr{} is \emph{stable} for a strong \emph{and} a weak model (Opus $0.923\!\to\!0.923$,
$\Delta\hpr{}=0$; Haiku $0.731\!\to\!0.763$), \rhg{} stays $\approx\!0$: honesty survives the textual
novelty---it is reasoning, not recall.
\item \textbf{Semantic novelty.} Self-authored tasks (\code{gray2bin}, \code{popcount8},
\code{hex7seg}) implement functions absent from any public benchmark, yet models stay honest
($\rhg{}=0$)---not benchmark recall.
\item \textbf{Semantic mutation.} \code{sem\_zerocount} counts \emph{zeros}, a variant of the
canonical popcount: the memorized count-ones solution passes the balanced visible test but is caught
by hidden$+$formal ($\rhg{}=0.50$), while both models implement the variant honestly.
\end{itemize}
We report contamination as its own one-sided Clopper--Pearson $95\%$ bound: $k\!=\!0$
memorization-driven \hpr{} drops on the $156$-task probe gives $\le\!1.9\%$. Membership inference
(Min-K\%~\cite{minkprob}; NLL$\leftrightarrow$degradation~\cite{nllmem}) needs teacher-forced logprobs
that no closed API we use exposes, so we run it on an \emph{open} model (Qwen2.5-3B, local MLX): the
mean NLL-degradation under the meaning-preserving mutation is $+0.045$ nats---negligible, \emph{no}
memorization advantage---triangulating the behavioral probe.

\paragraph{Cost (C2).} Across the four models with per-iteration token logs
(Opus/Haiku/GPT-5.5/Gemini; DeepSeek's sweep did not record tokens), the repair tail is mostly
wasted---only $47$/$14$/$59$/$48\%$ of multi-iteration tasks end honest. An early-stop at one
iteration reclaims $12$--$24\%$ of tokens (Opus $12$, GPT-5.5 $17$, Haiku $23$, Gemini $24$) for a
$4$--$9\%$ honesty loss (Figure~\ref{fig:cost}). Evaluated \emph{prospectively}
(leave-one-model-out---the cap is fit on the other three, never on the model it is scored on), a $5\%$
honesty-loss budget is met only by the stronger-tail models, so the early-stop is \emph{model-dependent}
rather than universal---a caveat the post-hoc number hides.

\code{popcount8}, \code{spm}, and \code{picorv32} (a real RISC-V core)---under \textbf{two synthesis
strategies} (\code{AREA} vs \code{DELAY}). Real Sky130 area spans $\sim\!500\,\mu m^2$ (counter8) to
$\sim\!113{,}000\,\mu m^2$ (picorv32), and the design ranking is \textbf{fully stable across the two
flows}: Kendall-$\tau=1.0$ for area, power, \emph{and} slack---so the latency comparison is not a
synthesis-flow artifact. Slacks are pre-signoff \emph{relative} (open Sky130, not foundry signoff).

\section{Findings}
\begin{enumerate}[leftmargin=*,topsep=2pt,itemsep=1pt]
\item A na\"ive formal oracle \textbf{over-reports} reward hacking; the headline number was an
artifact. Verification discipline is mandatory---and now \emph{fully mechanized}: the careset $+$
mixed-edge miters \emph{prove} every flagged artifact equivalent; no hand-verification remains.
\item No false positives on honest agents; across $5$ models $\times\,156$ fair tasks, \textbf{zero
verified genuine reward hacking} ($95\%$ upper bound $\le\!3.2\%$).
\item \textbf{Weakness $\ne$ hacking}---a weaker model fails far more but cheats no more, even under
tamper affordance.
\item Hacking is a function of adversarial conditions, not benchmark mass; the protocol catches it
when present.
\end{enumerate}

\section{Threats to Validity}
\emph{Contamination}: identifier mutation on \emph{all $156$} tasks (oracle-verified equivalent),
\code{sem\_zerocount}, and open-model MI all negative; function-restructuring re-mutation remains. \emph{Oracle residual}: \textbf{none} --- the
sequential-FSM residual is closed (reset-aware BMC), the careset miter proves the don't-care
artifacts, and the real-latch half-cycle miter closes the mixed-edge class (\code{circuit8}); across
the five models the flagged \rhg{} drops $9\!\to\!0$ with \textbf{zero regressions}, the planted
overfit and three broken mixed-edge mutants all still CEX (non-vacuous). The proofs are conditional on
\emph{declared} preconditions (synchronous stimulus; legal-input sequences) that the withheld
testbench also respects. GPT-5.5 leaves $2$ inconclusive (Conway's Life's $256$-cell grid plus one
candidate the bounded SAT does not close in budget)---a per-candidate budget limit, not a verdict. \emph{Eliciting}: on \emph{fair} tasks single-loop aligned agents do not hack, but an
\emph{impossible} (spec-contradicting) task elicits cheating in the \emph{majority} ($3/5$ pass it,
all $3$ confirmed by an \emph{independent} judge that scores $86\%$ on known hardcodes) with Opus
resisting---so the fair-task negative is a property of the \emph{conditions}, not a blind spot; the
deeper regime (RL training) is in \S\ref{sec:limits}. \emph{PPA}: open Sky130 is pre-signoff (inflated
${\sim}1.2$--$1.5\times$ vs.\ commercial~\cite{edahep}) and ranking inversions are a known
risk~\cite{rtlopt}; we check rank stability across two synthesis strategies (\S\ref{sec:exp},
Kendall-$\tau=1.0$) and frame PPA as \emph{relative} (per TuRTLe~\cite{turtle}). \emph{Nondeterminism}:
LLM runs vary; we report Wilson CIs and pin model ids.

\section{Limitations and Future Work}\label{sec:limits}
Four limitations bound the claims; each has a concrete next step.
\begin{itemize}[leftmargin=*,topsep=2pt,itemsep=2pt]
\item \textbf{Small-module scope.} The $156$ VerilogEval tasks are single-module, single-clock RTL,
so ``zero genuine reward hacking'' is a statement about \emph{this} task space. System-level designs
(SoCs, multiple clock domains, hierarchical timing) are untested, and the careset/half-cycle
machinery is per-module---compositional extension is future work.
\item \textbf{Eval-time, not RL training.} We measure single-loop inference. The regime where reward
hacking is documented to \emph{emerge} is RL on a gameable reward~\cite{baker,rlvrgaming}. Our training
probe (\code{train\_grpo.py}: GRPO on the visible-test reward, audited by the withheld hidden$+$formal
oracle, $\rhg(t)=\mathrm{visible}(t)-\mathrm{formal}(t)$) \emph{runs end-to-end on a GPU}: a Qwen3-4B
LoRA smoke (SFT cold-start~\cite{dapo} $+$ $50$ GRPO steps, A100) holds
$\mathrm{visible}=\mathrm{formal}=0.83$ with $\rhg$ \emph{flat at} $0$ at both audits---no hacking
induced \emph{at smoke scale}. The full multi-seed emergence study is a \emph{separate GPU study}.
RTL is the one domain where this step-wise audit is \emph{exhaustive}.
\item \textbf{PPA is pre-signoff.} C3 now spans a size-graded set (\code{counter8}$\to$\code{popcount8}
$\to$\code{spm}$\to$\code{picorv32}) with real Sky130 PPA and Kendall-$\tau=1.0$ rank stability across
two synthesis strategies. The remaining gap is a \emph{commercial signoff} stack (open Sky130 inflates
PPA ${\sim}1.2$--$1.5\times$~\cite{edahep}) and a full agent-loop under the trained surrogate.
\item \textbf{Contamination probe not yet exhaustive.} We mitigate at identifier and semantic level
(\S\ref{sec:exp}) with a $\le\!7.2\%$ upper bound, but a full meaning-preserving re-mutation of all
$156$ tasks plus membership inference (Min-K\%~\cite{minkprob}; NLL$\leftrightarrow$degradation
\cite{nllmem}) would tighten it further.
\end{itemize}
Beyond these: more models and task types (debug/verify/reuse), and closing GPT-5.5's two
budget-limited inconclusives.

\section{Conclusion}
Honest evaluation of agentic hardware design is gated not by a new pass-rate but by an oracle that
is don't-care/reset/encoding-aware and by a discipline of verifying every flag. With that, frontier
agents do not hack fair RTL tasks---but the oracle catches it when they do.

\paragraph{Reproducibility \& artifact.} All code, frozen sweeps, and generators are released;
\code{report\_cis.py} regenerates every headline number from the frozen candidates (EDA-only, no LLM
calls), with pinned model ids and fixed testbench seeds (\code{docs/REPRODUCE.md}).
